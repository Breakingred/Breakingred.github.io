    TCP/IP 分层管理通信传输

    客户端/服务器 备注
    应用层 如HTTP、FTP、DNS（域名系统）
    传输层 如TCP（传输控制协议）UDP（用户数据报协议） 分割HTTP报文，标记序号、端口号
    网络层 IP 增加通信地的MAC地址（网卡所属固定地址）
    链路层 网络 硬件

    URI和URL

    URI（统一资源标识符）用字符串标识某一互联网资源，URL（统一资源定位符）表示资源的地点。URL是URI的子集。

    HTTP的请求类型/方法

    HTTP协议用于客户端和服务器端的通信，客户端发出的请求类型包括以下几种：

    • GET

      请求获取资源

    • POST

      传输实体主体，注册、登陆、提交表单数据等操作都是用POST来完成

    • PUT

      传输文件，方法自身不带验证机制

    • DELETE

      删除文件，同样没有验证机制

    • HEAD

      获得报文头部，与GET 一样，只是不返回报文主题部分，用于确认URI 有效性及资源更新日期等

    • OPTIONS

      询问支持的方法
